# ScanSplitter 0.5.2

This patch release completes the multi-scan Quick workflow and improves photo
border accuracy on high-resolution scans.

## Multi-scan cropping

- Quick mode now offers separate **Crop Current** and **Crop All** actions,
  with the number of photos shown for each scope.
- **Crop All** waits for queued auto-detection to finish, then processes every
  scan with detected boxes serially and reports combined progress.
- Batch results automatically open in the **All** view, and the interface
  clearly shows how many uploaded scans have photos ready. This completes the
  workflow requested in
  [issue #13](https://github.com/Madnex/ScanSplitter/issues/13).

## More precise borders

- ScanSplitterv2 now refines contour rectangles against continuous grayscale
  edges near each proposed side. This prevents soft scanner shadows and halos
  from expanding crop boxes on high-resolution input.
- The refinement keeps the detected rotation, uses resolution-aware search
  bands, and preserves the original contour side whenever no trustworthy edge
  stands out.
- The new detector was verified against the original 5104×7016, 48 MB sample
  from [issue #15](https://github.com/Madnex/ScanSplitter/issues/15): all three
  photos remain detected and the refined crops exclude the surrounding white
  scanner background.

## Narrow sidebar layout

- The **Photo Date** editor now keeps its date field and clear action within
  narrow sidebars, with the **Apply** action moved to a full-width row below.
- Added regression coverage for the compact date-control layout.

## Validation

- Added frontend regression coverage for batch target selection, crop-action
  scope and counts, waiting for auto-detection, and compact metadata controls.
- Added detector coverage for high-resolution scanner shadows and the
  low-confidence fallback behavior.
