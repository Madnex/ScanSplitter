# ScanSplitter 0.5.1

This patch release improves the Quick workflow for larger scan batches and
makes repeated delivery to external photo services faster and safer.

## Quick workflow fixes

- Auto-detect now processes every file in a multi-file upload instead of
  cancelling earlier scans when the next scan starts. This fixes
  [issue #13](https://github.com/Madnex/ScanSplitter/issues/13).
- Large upload batches keep the full interface usable: scan tabs wrap inside a
  bounded, scrollable area without pushing the results panel off-screen. This
  fixes [issue #12](https://github.com/Madnex/ScanSplitter/issues/12).
- Valid album and naming-pattern changes now update actual export filenames
  immediately, matching the live preview without a separate Apply step. This
  fixes [issue #14](https://github.com/Madnex/ScanSplitter/issues/14).

## Delivery credentials and preferences

- Immich API keys and Nextcloud app passwords can optionally be stored in the
  operating system credential vault rather than project files or browser
  storage.
- Saved secrets are resolved only by the local backend, are never returned to
  the browser, and cannot be used or managed when ScanSplitter is running
  outside local mode.
- Saved connections can be reused, updated, or forgotten from the Delivery
  dialog. Systems without a supported credential vault continue to support
  one-time credential entry.
- The Delivery dialog remembers the last selected target, so recurring Immich,
  Nextcloud, or watched-folder workflows open with the preferred method.

## Maintenance

- Updated Python, frontend, and GitHub Actions dependencies, including fixes
  for dependency-audit findings.
- Added frontend regression tests for batch auto-detection, large tab layouts,
  automatic naming, and preferred delivery targets, plus backend coverage for
  secure credential handling and local-mode enforcement.
