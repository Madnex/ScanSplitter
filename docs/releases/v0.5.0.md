# ScanSplitter 0.5.0

This release expands ScanSplitter from a quick scan-splitting utility into a
non-destructive workflow for larger photo-archiving projects.

## Highlights

- Create persistent projects, upload scans in bulk, and resume work across
  sessions.
- Review only uncertain detections with confidence flags, keyboard navigation,
  background processing, progress reporting, and cancellation.
- Add archival dates, places, GPS coordinates, captions, people, events,
  albums, and manual front/back pairings without modifying original scans.
- Preview and apply conservative deskew, color/fade correction, and optional
  non-generative 2× upscale to exported copies.
- Export approved photos as JPEG or lossless PNG. PNG export was requested in
  [issue #3](https://github.com/Madnex/ScanSplitter/issues/3).
- Create optional PNG/TIFF masters, organized folders, and JSON/CSV provenance
  manifests.
- Deliver finished photos directly to Immich, Nextcloud WebDAV, or a watched
  local folder. Immich supports a least-privilege API key with only the
  `asset.upload` permission.

## Reliability and privacy

- GPS metadata remains opt-in for every export or delivery.
- Original project scans are never modified by metadata, restoration, or
  export operations.
- Network-delivery credentials are request-scoped and are not stored in project
  files.
- Improved path containment, upload validation, decompression limits, model
  checksum verification, duplicate-name handling, and structured job errors.
- Added OpenCV 5 compatibility and expanded backend, frontend, packaging, and
  real-world workflow coverage.

Built-in OCR and automatic dust/scratch repair were intentionally left out of
this release after real-world testing showed that their results were not
reliable enough for archival use.
